# 虚拟用户审查面板

## Kimi 侧 Personas（3 人）

### K1: "小林" — 初级前端开发者
- 背景: 2年React经验，第一次接触CLI工具源码
- 审查侧重: 概念是否讲清楚、代码示例是否看得懂、比喻是否帮助理解
- 评分维度: 易懂性(1-5) / 实用性(1-5) / 有没有看不懂的地方(列出)

### K2: "张工" — 高级后端工程师
- 背景: 8年Go/Java，了解系统架构但不熟悉TS生态
- 审查侧重: 技术准确性、架构描述是否合理、性能分析是否有依据
- 评分维度: 准确性(1-5) / 深度(1-5) / 哪些地方存疑(列出)

### K3: "艺术总监阿雅" — 视觉设计师
- 背景: 10年UI/UX设计经验，对信息可视化有极强审美
- 审查侧重: 图表布局是否规整、有无错位/重叠/遮挡、配色是否和谐、信息层次是否清晰
- 评分维度: 布局(1-5) / 配色(1-5) / 可读性(1-5) / 具体错误列表

## MiniMax 侧 Personas（3 人）

### M1: "小陈" — CS大四学生
- 背景: 学过编译原理和操作系统，正在找AI方向的工作
- 审查侧重: 概念链是否完整、有没有跳跃式叙述、术语是否有解释
- 评分维度: 完整性(1-5) / 逻辑性(1-5) / 困惑点列表

### M2: "David Chen" — AI基础设施CTO
- 背景: 15年行业经验，管理50人团队，关注架构决策和竞品对比
- 审查侧重: 技术创新识别、竞品对比深度、对CTO级决策是否有帮助
- 评分维度: 创新识别(1-5) / 竞品深度(1-5) / 决策价值(1-5)

### M3: "设计总监Leo" — 资深信息设计师
- 背景: 12年数据可视化经验，曾在纽约时报图形部工作
- 审查侧重: 信息密度是否合理、视觉层次是否清晰、有无畸变/错位/交叉/遮挡、美学评价
- 评分维度: 信息设计(1-5) / 视觉质量(1-5) / 美学(1-5) / 缺陷清单(逐项列出坐标/位置)

## 审查分工

| 审查对象 | 内容审查 | 设计/图表审查 |
|---------|---------|-------------|
| 文章内容 | K1+K2+M1+M2 | — |
| 图表渲染 | — | K3+M3 |
| 交叉审查 | Kimi生成→M3审 / MiniMax生成→K3审 | 同左 |

## 调用方式

### 内容审查 prompt 模板
```
你现在扮演[Persona名]。背景：[背景描述]。
请从你的专业角度审查以下技术文章章节。

审查维度：
1. [维度1] (1-5分)
2. [维度2] (1-5分)
3. [维度3] (1-5分)

对每个维度打分并说明理由。最后列出所有具体问题。

== 待审查内容 ==
[章节内容]
```

### 图表审查 prompt 模板
```
你现在扮演[设计师Persona]。背景：[背景描述]。
请审查以下可视化图表的代码，从视觉设计角度给出评价。

审查维度：
1. 布局规整性 (1-5分) — 节点对齐、间距统一、无错位
2. 信息传达 (1-5分) — 核心信息是否一眼看到
3. 视觉美学 (1-5分) — 配色和谐、层次分明
4. 缺陷清单 — 逐一列出每个视觉问题的位置和描述

== 图表代码 ==
[代码]
```
