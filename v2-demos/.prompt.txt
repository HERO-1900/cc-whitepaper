### C. 五代范式演化（重要！请用视觉强烈的"演化阶梯"或"时间轴+卡片"表达）

- 第 1 代 Pure Prompt（2022 年前后）：写 prompt → 发 → 收 → 结束。无工具、无循环、无状态。
  通俗类比：发短信叫外卖，对方回一条就完了。
- 第 2 代 Tool Use（2023）：function calling、MCP 工具调用。
  通俗类比：外卖平台接入了地图、库存、支付，但还是无状态单轮。
- 第 3 代 Multi-Agent / Orchestration（2023-2024）：CrewAI、AutoGen、LangGraph，工程师设计 agent 协作关系。
  通俗类比：物流公司有调度员/骑手/客服，但流程靠工程师事先画好。
- 第 4 代 Harness（2025 至今，**当前位置**）：精心设计 agent 运行的环境本身——进度文件、特性清单、启动脚本、lint 规则、反馈循环、垃圾回收……
  通俗类比：赛车的安全带+仪表盘+赛道规则。
  代表实践：Young 的 initializer + coding agent；Lopopolo 的 AGENTS.md-as-table-of-contents；LangChain 的 generator-evaluator 循环。
- 第 5 代 端到端 RL（行业争议中）：Noam Brown 等认为 harness 是打补丁，会被更强的模型自然消灭。但 Rajasekaran 反驳："the space of interesting harness combinations doesn't shrink as models improve. Instead, it moves."

### F. 行业争议（核心冲突，请用"VS 双栏对峙"视觉表达）

**怀疑阵营：**
- Chayenne Zhao 的"新名旧物"批评：每隔几个月就新造一个词，同一件事换不同包装
- Noam Brown 的 Bitter Lesson 立场：harness 是"在 pre-training 不够强时打的补丁"
- METR 研究：专业 harness 相比 basic scaffold 的优势 negligible
- Scale AI SWE-Atlas：harness 选择产生的性能差异在实验误差范围内
- HumanLayer 的 Kyle 反直觉发现：Opus 4.6 在 Claude Code 官方 harness 下排名 #33，在 LangChain 重设的 harness 下排名 #5——可能是「模型对自己 harness 过拟合」

**支持阵营：**
- 分层是软件工程的基本方法，OS 在应用层之下，没人认为分层是"拐杖"
- Rajasekaran 的实证：从 Opus 4.5 升级到 4.6 时，sprint 结构被简化、context reset 被移除，但 planner 和 evaluator 保留。"the space of interesting harness combinations doesn't shrink as models improve. Instead, it moves."
- NLAH 的发现：harness 作用区域在"模型差一点但做不到"的边界——边界往外移，harness 在新边界继续有价值
- LangChain 实证：只改 harness 让同一模型在 Terminal Bench 2.0 提升 13.7 百分点

swyx 的总结："**Big Model vs Big Harness**"——"You can engineer your way above the Bitter Lesson, and harnesses can survive even reasoning paradigm changes."

**作者判断**：harness engineering 是真实工程需求的系统化表达，不是全新发明，但值得独立命名；它是成熟软件工程原则（分层、约束、反馈循环、渐进增量）在 LLM agent 场景的具体化。

## 2. V2 范式核心原则（给你的设计指南）

### 文字与"图表元素"必须共生
不要把所有"卡片/数字大字号/时间线"堆在文末。每一段的解释文字旁边/下面/穿插着属于它的视觉表达。例如：
- "Trivedy 在 2025 年 9 月 23 日…" → 旁边就是时间线上对应那一格的高亮
- "OpenAI 5 个月写了 100 万行代码" → 这段文字旁边就是"100 万 / 5 月 / 1500 PR / 6h+"四个大数字
- "三家对比" → 三栏对比卡片就在介绍三家的段落正下方
- "Harness 六要素 × CC 章节" → 一个视觉化映射表，章节号是 chip 形式

### 视觉层次要丰富
- 大字号引言（pull quote）
- 数字大字号 stat（48~72px）
- 章节"段位"标签（"01 · 引子"这种）
- 双栏对峙（VS 视觉）
- 时间线（横向或竖向都可，关键节点要有日期和高亮节点）
- 卡片网格（论文卡片、对比卡片）
- 引用块（金句单独成块）
- 通俗类比的特殊视觉容器（带"💡"图标的提示块）

### 移动端响应式
- 桌面 ≥1024px：多栏布局
- 平板 768-1024px：两栏
- 手机 <768px：单栏，时间线变竖向，对比表变堆叠卡片

## 4. 主题切换按钮（必须包含）

页面右上角放一个简单的主题切换按钮（"暗 / 亮"），点击切换 `<html>` 的 `data-theme` 属性。代码示例：

```html
<button id="themeToggle" style="position:fixed;top:16px;right:16px;z-index:999;background:var(--cc-bg-elevated);color:var(--cc-text-primary);border:1px solid var(--cc-border-default);border-radius:8px;padding:8px 12px;cursor:pointer;font-size:13px;">切换主题</button>
<script>
document.getElementById('themeToggle').addEventListener('click', () => {
  const html = document.documentElement;
  const cur = html.getAttribute('data-theme') || 'dark';
  html.setAttribute('data-theme', cur === 'dark' ? 'light' : 'dark');
});
</script>
```
