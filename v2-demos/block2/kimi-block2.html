在这个群雄并起的格局中，Claude Code 选择了一条独特的道路——

不做 IDE 插件 → 不做 Web 应用 → 而是在终端里构建一个完整的 AI 智能体运行时

IDE 是程序员写代码的专用编辑器，类似于设计师的 Photoshop。 （终端是那个黑底白字的命令行窗口）

唯一一个可以在纯远程服务器上工作、可以完全通过键盘操作、可以与任何编辑器无缝集成的 AI 编程助手。

代价是：它必须自己解决界面渲染、文件管理、进程隔离等一系列在 IDE 环境中本来"免费"的问题。这本书要解剖的，正是这个雄心勃勃的工程选择背后的全部技术细节。

行业对这类系统的认知经历了三个阶段的演进

2023-24
提示工程
Prompt Engineering
2025
上下文工程

2026
套控工程
Harness Engineering

LangChain 创始人 Harrison Chase 提出的三层架构

Model
Runtime
Harness

harness（套控层）的设计差异决定了产品表现的差异。

💡 通俗理解
"套控"就是给 AI 戴上"马具"——既要让它能拉车干活（工具能力），又要套上缰绳保证它不会乱跑（安全边界）。

本书解剖的 Claude Code，正是一个 harness 设计的完整案例。

"群雄并起"不只在公司之间发生——它已经发生在 Claude Code 内部。
一个 CLI 工具里就有 12 个内置 Subagent。
