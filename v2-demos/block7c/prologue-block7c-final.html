BLOCK 07C

内部组织

看完了系统怎么向外延展，让我们把视角收回来——看系统自己长什么样。
它的内部组织有两个层面：可配置的"软组织"，和模块之间的"骨架"。

Claude Code 白皮书 V2 · 第 9/11 块
OS 类比锚点
/etc/ 配置 | 设置系统（5 层 + 4 子层） | 从企业策略到用户偏好的九层配置合并，就像 /etc/ 下的系统配置覆盖用户 ~/.config
通俗理解
配置系统就像穿衣服的层次——贴身内衣 = 默认配置 → 衬衫 = 项目配置 → 外套 = 用户配置 → 防弹衣 = 企业策略。外层覆盖内层，但防弹衣（企业策略）最高优先级，谁也脱不掉。

配置是系统的"软组织"——让我们看看这件衣服具体长什么样

配置文件完整地图

全局 · 项目 · 工作区三层配置树

~/.claude/ （全局配置）
⚙️ settings.json 核心
全局用户设置，包含默认权限模式、主题偏好、编辑器配置等。优先级低于项目级设置。
⌨️ keybindings.json 交互
自定义键盘快捷键映射，支持 chord 绑定和多层快捷键覆盖。
🔐 credentials.json 敏感 · 勿提交
⚠️ 请勿提交 git
存储 API Key、OAuth Token 等敏感凭证。已自动加入 .gitignore 保护。
🧠 memory/ 持久
用户级记忆存储目录，包含 user_role.md、feedback_*.md、project_*.md 等记忆文件。
📊 projects/ 统计
各项目统计信息、使用数据、项目级缓存文件存储目录。
✓ todos/ 任务
全局待办事项存储，跨项目的任务追踪数据。
📈 statsig 遥测
功能开关和实验配置缓存，用于 A/B 测试和灰度发布。
.claude/ （项目配置）
⚙️ settings.json 项目
项目级设置，覆盖全局设置。团队共享的配置标准放置于此。
🔧 settings.local.json 本地
个人本地覆盖，不会被 git 跟踪。用于开发者的个人偏好。
📝 commands/*.md 自定义
项目自定义斜杠命令定义文件，每个 .md 对应一个 /command。
📋 CLAUDE.md 最高
项目级系统指令文件，最高优先级配置，定义项目上下文和规范。
工作区配置
📄 <dir>/CLAUDE.md 目录级
任意目录下的 CLAUDE.md，作用于该目录及其子目录，优先级最高。
🚫 .claudeignore 忽略
控制 Glob/Grep 搜索范围，类似 .gitignore 但专用于 Claude Code 的文件索引。

配置决定系统的"脾气"，那决定系统的"骨骼"是什么？
答案是模块之间的依赖关系

Tool.ts 是 Claude Code 工具系统的核心枢纽，被 43 个工具实现所依赖。它定义了工具接口、执行上下文和权限检查机制。
Tool.ts 是最大的枢纽——这也是为什么工具系统是 Claude Code 的半条命

配置层（软）+ 依赖层（硬）共同构成系统的内部肌理——
这也是为什么 Claude Code 在不同环境下会表现出不同行为，但核心结构始终稳定。
