外部接口

Claude Code 不是一座孤岛——它有三种对外延展面。标准化协议（JSON-RPC）接入外部能力，就像驱动让 OS 操控不同厂商的硬件。在系统运行的 27 个关键节点注入自定义逻辑，就像 Linux 的可加载内核模块。第三方扩展的安装、验证、沙箱化执行——和 OS 的包管理器做同样的事。

💡
通俗理解
MCP 就像 MacBook 的 USB 转接口——MacBook 只有 Type-C 口，想接鼠标键盘投影仪都需要转接器。MCP 就是 Claude 的万能转接器，让它能连接各种外部工具和服务。

转接器有 8 种规格，每种适合不同场景——让我们从"我该选哪种"的角度切入。

MCP 传输方式对比

8 种 MCP 传输方式的五维评估

MCP（Model Context Protocol）定义了 Claude 与外部工具通信的协议层，但数据如何在两者之间流动，有 8 种不同的传输方式可选。它们在延迟、吞吐量、安全性、实现复杂度和连接可靠性五个维度上各有优劣。

MCP 让 Claude 连接外部工具，但有时候你想在 Claude 内部的某个时刻插一段自定义逻辑——这就是 Hooks。

💡
通俗理解
Hooks 就像快递柜的短信通知——快递到了自动通知你 = 事件钩子。你可以设置"到了通知我"或"直接放门口" = 自定义 Hook 行为。系统在关键时刻自动触发你预设的动作。

MCP 是"连别人"，Hooks 是"插自己"，那么"装第三方扩展"呢？答案是 Skills + 插件。

扩展不是一个机制，而是三种不同尺度的接口——一种连外部（MCP）、一种插内部（Hooks）、一种装第三方（Skills+插件）
