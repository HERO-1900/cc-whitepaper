核心反转：这不是聊天助手

你打开的是一个拥有 1,884 个 TypeScript 文件 的工程系统。它有自己的进程调度器、安全沙箱、文件版本管理、多实例协调器、遥测管线、插件生态。

一个为 AI 智能体设计的操作系统。

当然，"操作系统"是帮助理解的类比，不是精确的技术定义——Claude Code 运行在 Node.js 之上，本身仍是一个用户态应用程序。但它承担的职责（资源调度、安全隔离、生命周期管理、扩展机制）确实和操作系统高度同构。

四层纵深防御

安全不是单点检查，而是纵深防御。Claude Code 构建了四层防御体系：权限层（十步状态机）、沙箱层（seatbelt/bwrap 进程隔离）、命令预检层（Bash AST 分析）、企业策略层（bypass-immune 规则）。

三层威胁向量（用户输入、AI 生成、外部工具）在每个防御层都要接受检查。

权限模型对照矩阵

Claude Code 定义了 6 种权限模式：全自动、全确认、只读、最小权限、企业锁定、完全离线。每种模式对各类工具的"自动/确认/禁止"行为都有明确定义。

这是一个 6×N 的决策矩阵，企业管理员可以通过九层配置体系远程控制每一台开发者机器上的行为。

MCP：标准化的设备驱动

MCP（Model Context Protocol）就像操作系统中的设备驱动层。通过标准化的 JSON-RPC 协议，Claude Code 可以接入外部能力：GitHub、数据库、浏览器、自定义工具……

不同传输方式在延迟、吞吐、安全、复杂度、可靠性五个维度各有取舍。
