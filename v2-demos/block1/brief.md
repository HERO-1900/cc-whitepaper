## 0. 给生成 Sub-Agent 的硬约束（最优先读）

**MUST**:
- 输出唯一一份完整 HTML 文件（含 `<!DOCTYPE html>`、`<head>`、`<style>`、`<body>`），无外部依赖
- token 引用：仅使用 `var(--cc-*)`，不得在本块内重新定义任何 `--cc-*` 变量

- 不得调用任何 JavaScript 框架或 CDN，纯 CSS + 极少量必要 vanilla JS（仅主题切换 + 可选 claude 光标闪烁）

### 3.3 hero footer 组件 spec（OQ4 — 全书首次定义，后续 8 块必须复制）

Block 1 是第一个使用 hero footer 的块，本节定义的 spec 会被签署为全书 11 块的标准。**未来块不得改 spec**。

```
hero-footer 组件结构：
[ Block 编号 chip ]  [ 块名 ]  [ 一行 sub-title ]
       L              块名居中     右侧或下方
```

字段定义：
- **块号 chip**：`0 / 序章 · 01` 字号 `--cc-font-size-sm`，背景 `--cc-color-surface-secondary`，圆角 pill
- **块名**：「Hero 一击 · 引擎盖悖论」字号 `--cc-font-size-2xl` ~ `3xl`，颜色 `--cc-color-text-primary`
- **sub-title**：「全书第 0 拍 · 5 秒钩子」字号 `--cc-font-size-sm`，颜色 `--cc-color-text-secondary`
- **位置**：每块顶部，紧邻 marginal label 之下、hero 内容之上
- **留白**：`--cc-space-8` 上下，`--cc-space-6` 左右

后续块复制时只需替换块号 / 块名 / sub-title 文字内容。**spec 不许改**。

- L5 / L7 段落字号：`var(--cc-font-size-lg)` ~ `xl`，`var(--cc-leading-relaxed)`
