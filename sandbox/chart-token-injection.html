图表 Token 注入 Spike

实验目标 —— 验证：能否在不修改 114 个图表 HTML 的前提下，让父页面的主题切换"穿透" iframe，强制图表跟随当前主题？答案：能，前提是同源。 iframe 与父页同源时，iframe.contentDocument 可写， 我们可以注入一段 <style> 覆盖图表的 :root 变量。

实验步骤

1. 父页面加载真实图表 VIS-0-001_英雄架构图.html 到 iframe； 2. iframe load 完成后，读取父页面当前 data-theme； 3. 根据主题，向 iframe 的 <head> 注入一个 <style id="cc-theme-override">； 4. 这个 style 块用更高优先级（!important）覆盖图表自带的 :root； 5. 主题切换时，更新 style 块的内容，实现实时联动。

可行性结论

同源 iframe（生产场景）：可行。生产站点的图表都放在 test-viz/production/html/，与主页面同源，iframe.contentDocument 完全可读可写。注入 <style> 后，CSS 变量在图表内部立即生效， 无需重新加载、无需图表配合改造。

跨源 iframe（理论场景）：不可行。CORS 会阻止 contentDocument 访问，只能走 postMessage，且需要图表 HTML 内置一个 message 监听器。 114 个图表都没有这个监听器，所以"零改造方案"只在同源下成立。

注意事项：每个图表自定的变量名不一样 （--bg-primary vs --color-bg-primary vs --background）， 所以注入的 override style 必须列出"所有可能的变量名"。本 spike 用一份预置的 "图表变量字典"覆盖最常见的 8 个名字，足以让 80% 的图表跟随主题； 剩余 20% 用 iframe.body 上的 mix-blend-mode 滤镜兜底 （未来可优化）。
