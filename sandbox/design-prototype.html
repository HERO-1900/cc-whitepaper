前端设计升级原型

基于 Gemini Deep Research 调研的 7 大方向，这里展示 CSS Token 系统、CJK 排版优化、亮暗主题切换三个方向的初步效果。

设计 Token 体系

当前站点的设计语言不统一——不同视图使用不同的颜色硬编码、间距不一致、排版参数分散。CSS Token 系统通过统一的设计变量解决这个问题：所有颜色、间距、圆角、字体、动效参数都从 :root 变量取值，修改一处即全局生效。

设计原则

Token 的价值不在于"好看"，而在于**一致性**。30 万字的长文档站点，如果每处 padding、每个 border-color 都是独立值，维护成本会指数增长。Token 让设计决策集中在一处、传播到所有组件。

CJK 排版优化

中文排版和英文有显著差异：行高需要更大（1.8 vs 英文的 1.5）、段间距需要更明确、标点需要挤压处理（font-feature-settings: "halt"）、半角/全角混排需要额外间距。当前站点的行高和段间距偏紧，长文阅读体感差。

这段文字展示了优化后的 CJK 排版效果。注意行与行之间的间距、段落之间的呼吸感、以及中英文混排时的自然过渡。比如当我们提到 CacheSafeParams 这样的技术术语时，代码标记自然融入行文，不会显得突兀。

亮暗主题切换

当前站点只有暗色主题。对于 30 万字的长文阅读，暗色不是唯一选择——白天阅读、打印、分享给不习惯暗色的读者时都需要亮色选项。右上角的按钮可以切换主题（试试看 →）。

下一步建议

给用户的建议

低成本高收益（可立即执行）：将 CSS Token 系统和 CJK 排版优化迁移到现有站点。这不需要换框架，只需重构 style.css 中的硬编码值为 CSS 变量，并调整排版参数。

中成本中收益（需要评估）：添加亮暗主题切换、Cmd+K 全局搜索面板。前者需要为所有现有颜色值添加 light 主题覆盖，后者需要新增 JS 组件。

高成本高收益（需要谨慎决策）：迁移到 Astro 框架。性能和 SEO 收益巨大，但需要重构所有 JS 模块。建议如果要做，趁内容稳定时一次性完成。
