**缺陷:**

1. **布局/错位** — `.pattern-card.featured { transform: scale(1.02) }` 使卡片宽度超出grid容器的 `calc(100% + 20px - 4px)`，在固定1000px max-width下右侧溢出或挤压邻卡；hover状态 `scale(1.02) translateY(-4px)` 组合叠加更明显
2. **布局/溢出** — `.number-badge { top: -10px; left: -10px }` 负定位使徽章超出 `.pattern-card` 上/左边界，在紧邻边界时可能被父容器裁切
3. **美学/一致** — Card 2比其余卡片多一个 `.warning-box` 元素，导致高度不一致；其余卡片高度参差不齐
4. **代码/语法** — `.card-2 .warning-box` 背景 `rgba(239, 68, 68, 0.1)` 使用红色（警告色），但Card 2主题色是绿色，配色语义冲突
5. **响应式/错位** — 768px以下 `.card-grid` 切换为单列，但 `.pattern-card.featured { transform: none }` 取消了scale，这会使原本等高的featured卡突然高度收缩

---

**修改建议:**

1. 移除 `.pattern-card.featured { transform: scale(1.02) }`，改用 `border: 2px solid` 或 `box-shadow` 强调，避免尺寸变化撑破布局
2. 将 `.number-badge` 负定位改为 `top: 12px; left: 20px` 正定位在卡片内部，或给 `.pattern-card` 加 `overflow: visible`（需配合父容器 `overflow: hidden` 策略）
3. 统一卡片高度：给 `.pattern-card` 加 `min-height` 或使用 CSS Grid `align-content: start` 配合统一结构；Card 2的warning-box改为独立section而非嵌入卡内
4. Card 2 warning配色从红色改为橙色/黄色（`#f97316`系），与绿色主题一致
5. 响应式媒体查询中移除 `transform: none`，或给 `.pattern-card` 统一加 `min-height` 防止单列时高度骤变
