缺陷:

1. **node-sprite 高度未约束** (line ~130)
   - 内部含 `.duck-frame` (top:70px, 高≈65px) 和 `.details` (margin-top:75px)
   - 由于 node-sprite 无 position:relative/height，其绝对定位子元素以 diagram 为参考而非自身
   - 导致 click 事件 offsetX/Y 计算错误，影响心形粒子生成位置

2. **稀有度条与属性区块纵向未对齐** (lines 200-250, 252-280)
   - rarity-section: left:200px; stats-section: left:560px
   - 状态节点从 left:20→920px 横向展开，但下方两辅助区块偏左，与主轴视觉脱节

3. **tick-indicator 与 stats-section 水平重叠** (lines 185-190, 265)
   - tick-indicator: right:20px, top:320px → 实际 left≈980px
   - stats-section: left:560px, top:320px, width:300px → 实际 left≈560px
   - 两者 y 坐标相同(320px)，tick-indicator 的 left:980px 与 stats 区块(560-860px)存在40px间隙，逻辑上不重叠但视觉上分散

4. **idle-sequence 与 stats-section 垂直重叠** (lines 282-290, 265)
   - idle-section: top:435px, height≈22px → 底部在 457px
   - stats-section: top:320px, padding:16px → 底部在 320+16+4×32=464px
   - 两者在 435-457px 区间重叠约22px

5. **duck-frame 样式定义在 node-sprite 之后但使用相同选择器** (lines 100-108, 130)
   - `.duck-frame` 的 left:20px, top:70px 相对于 node-sprite 定位
   - node-sprite 无明确高度时，绝对定位子元素以 diagram 为包含块
   - 建议加 `position:relative` 和 `min-height:140px` 到 node-sprite

6. **Anti-cheat 与 Hash Formula 垂直间距过紧** (lines 172-180)
   - hash-formula: top:130px
   - anti-cheat: top:130px
   - 两者并排于同一水平线，间距仅靠 gap:8px 撑开，视觉上略显拥挤

7. **idle-sequence 水平溢出风险** (line 282)
   - 15个 step × 22px + 14个 gap×3px = 330px + 42px = 372px
   - container width:1100px，idle-section left:560px → 右侧边界在 932px
   - 372px 宽度可容纳，但 400px width 设置偏大

8. **图例 Legend 与 stored-box 底边对齐** (lines 295-305, 176-180)
   - stored-box: top:20px, 估计高度约80px → 底部在 100px
   - legend: bottom:20px → 相对于 container 底部定位
   - container padding:40px，legend 实际 top: 480-40-20=420px
   - stored-box 与 legend 纵向跨度大(100px vs 420px)，中间空白过多
