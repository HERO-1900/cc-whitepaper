**缺陷**

1. **第 171 行 `data-height="30%"` 语义反** — Before 条 height:100%，After 条 height 应从 0% 增长至 30%（表示相对基准减少 70%），但 JS 读取 `data-height` 并设为目标高度，方向正确；然而 `data-val="-70%"` 标签动画在动画完成后才显示，实际视觉上两 bar 并排时 After 条明显矮于 Before，语义无误。**实际无缺陷，属误判。**

2. **第 194 行 SVG 箭头路径 `M150 90 L170 90`** — 起点 x=150 落在第一组末尾（Round 3 圆点 cx=140）与第二组起点（Predict 圆点 cx=180）之间偏左侧，且 marker-end 引用 `#arrow` 定义于 `<defs>` 中，方向由 `orient="auto"` 决定，实际箭头可能指向略偏；此外此箭头与两组之间的"减少 API 往返"语义关联不够直观，两组之间无视觉连接暗示对应关系。

3. **第 207–215 行 gauge 数值动画使用 `setInterval`** — 其他所有数值动画均用 `requestAnimationFrame` + cubic-bezier 缓动，仪表盘数值（0→90）却用 setInterval 每 20ms 线性 +2，与整体风格割裂；且固定步长在 90 结束时刚好为 45×20ms=900ms，与其他动画时长不一致。

4. **第 107 行 `.cards-col::before` 时间轴左偏移** — 桌面端 left:24px 与卡片内 `.card-num`（无明确 left 定位，纯流式布局）的视觉对齐依赖字体度量，非可靠对齐；移动端改为 left:14px，但卡片序号仍为 24px 起始，线条与序号明显错开。

5. **第 144 行 `.bar.after::after` 的 opacity 过渡** — `transition:opacity .4s ease .8s` 在 bar 高度动画（1.2s）完成后 0.8s 才渐显，总计 2s 才稳定；期间数值标签闪烁感明显，且与其他元素的稳定时间不一致。

**修改建议**

1. **箭头对齐** — 将第三卡 SVG 中两组圆点水平对齐至同一 y=110，第一组（Round 1–3）用灰色虚线，第二组（Predict/Result）用绿色实线，中间用一条垂直虚线或斜箭头表示"预测压缩"关系，使因果关系一目了然。

2. **统一缓动风格** — 将 gauge 数值滚动改为 `requestAnimationFrame` + easeOut 缓动，时长取 1400ms 与 `stroke-dashoffset` 动画对齐，实现同步稳定。

3. **时间轴对齐修正** — 给 `.card-num` 添加 `padding-left` 或显式定位，使序号圆点始终与 `.cards-col::before` 线条的视觉中心对齐；移动端尤其需校准。

4. **数值标签动画提前** — 将 `.bar.after::after` 的 `transition-delay` 从 `.8s` 改为 `1.0s`（height 动画完成前 0.2s 出现），配合 `transition-duration .3s ease`，使标签在 bar 停止移动后快速稳定。

5. **仪表盘质感** — 考虑将仪表盘填充改为渐变+内发光（`filter:drop-shadow` 已在用），或增加细分割线刻度，提升精致度匹配整体设计语言。
