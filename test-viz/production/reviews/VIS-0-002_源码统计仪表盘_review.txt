**缺陷:**

1. **Donut图表中心孔定位错误** — `renderDonut()` L199: `hole.setAttribute('cx', centerX + '%')` 中 `centerX=50` 数值与 `%` 单位混用，导致中心孔实际渲染在错误位置（SVG viewBox默认为100单位坐标系，数值50配合%应为0.5，但代码直接拼接为"50%"）

2. **Donut图表中心文本定位** — L207-214: 同样问题，`text-anchor="middle"` 配合 `x="50%"` 但与前面 path 绘制的弧度计算坐标系统不一致

3. **依赖图表Y轴坐标混用** — `renderDeps()` L249: `label.setAttribute('y', y + 5 + '%')` 将数值 `y`（如20, 50, 80）与字符串 `'%'` 直接拼接，SVG解析为 `25%` 而非预期的 `y%` 偏移量

4. **测试覆盖率环形图transform计算错误** — `renderTests()` L274: `transform: rotate(-90 ${cx*4} ${cy*2.5})` 使用硬编码乘数（4和2.5）与百分比坐标 `cx + '%'` 不匹配，导致圆环起始位置计算错误

5. **Histogram条形图高度计算不合理** — L137: `(d.pct / 40) * 80` 对应 36% 档位计算出 72% 高度，但 0.5% 的档位仅 1% 高度，视觉对比度过大，建议改为 `(d.pct / 36) * 70`

6. **热力图容器高度未约束** — `.chart-body#heatmap` 依赖 `flex: 1` 但父级无固定高度，导致 `.heatmap-grid` 的 `height: 100%` 参照物不明确

7. **水平条形图标签可能溢出** — L157: `label.setAttribute('x', '1%')` 对长文件名（如 `permissions.ts`）的8-9字符、9px字号而言，1%外边距约16px，在窄视口下易与条形重叠

---

**修改建议:**

1. 统一SVG坐标系：使用 viewBox="0 0 100 100" 并全程使用数值，或全程使用百分比单位
2. 修复Donut中心孔：改为 `cx="50" cy="50" r="21"` (数值单位匹配viewBox)
3. 修复依赖图Y轴：改为 `y="${y}%"` 或使用 transform 属性精确定位
4. 修复环形图transform：使用 `transform-origin: center` 配合 CSS 而非硬编码坐标计算
5. 调整直方图高度映射：`const maxPct = Math.max(...data.map(d => d.pct)); height = (d.pct / maxPct) * 70;`
6. 为 `.chart-body` 添加 `min-height: 0` 防止flex子项溢出
7. 增加条形图左侧外边距或截断过长文件名
