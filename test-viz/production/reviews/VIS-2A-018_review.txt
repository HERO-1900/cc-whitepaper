**缺陷:**

1. **行165** — `opacity: 0.15` 使MAIN外层晕圈几乎不可见，建议提升至 `0.3~0.4` 或改用 `stroke` + `stroke-width` 描边方案
2. **行163** — MAIN标签 `y=145` 与节点圆心 `y=90` 间距55px；子Agent标签间距仅40px；视觉重心偏下，不对称
3. **行~82~86** — `repeating-linear-gradient` 模拟虚线在Retina屏和不同宽度legend-line下会失真，应改用 `stroke-dasharray` + 真实 `<line>` 元素
4. **行~260** — `createParticle` 每400ms/500ms/600ms持续创建新 `<circle>` DOM节点，`particleGroup` 永不清理，动画运行久后内存累积
5. **行~240** — `moveTooltip` 只修正右/下边界，未处理左/上边界溢出；当光标贴近容器左上角时 tooltip 会跑出容器
6. **行~200** — 节点hover时 `<g class="node">` 整体scale，但子 `<text>` 元素独立于transform之外，导致文字与圆脱节

1. 统一MAIN与子Agent的标签间距均为40px，调整MAIN标签y至135
2. Legend虚线改用 `<line stroke-dasharray="4 4">` 替代gradient hack

4. tooltip边界检测补充左/上判断：
   ```javascript
   if (x < 0) x = e.clientX - rect.left + 16;
   if (y < 0) y = e.clientY - rect.top + 16;
   ```
5. 将节点hover的transform移至外层 `<g>` 以保证文字同步变换；或改用CSS `transform-origin: center` 作用于整个 `<g>`
