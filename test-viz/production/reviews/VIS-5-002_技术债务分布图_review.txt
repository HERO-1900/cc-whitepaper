### 缺陷（按严重程度）

1. **[严重] Tool Interface 标注越界**  
   标注文字 `y=128/140` 落在模块内部 `y=180~260`，且引线端点 `y=150` 也在模块内部。标注完全被模块遮盖。  
   坐标：`path d="M 590 180 L 590 150"` + text `y=128/140`

2. **[严重] 引线颜色与箭头断裂**  
   `--color-gray: #555` 经 `opacity:0.6` 后实际显示约为 `#222`，但 `#arrowhead` 硬编码为 `#555`（不透明），箭头比线深两个色阶。

3. **[中等] 对角线 `Type Definitions→MCP` 掠过 Tools Implementation 角落**  
   路径 `(440,100)→(560,220)` 在 `y=180` 处 x≈504，刚好擦过 Tools Implementation 左边缘（370~510），视觉上像穿过模块。

4. **[中等] QueryLoop 标注引线断头**  
   `path d="M 360 100 L 400 100"` 端点 `x=400`，但文字 `x=430`，间距30px无连接。

5. **[中等] 权限模块有标无引**  
   标注文字 `x=520 y=415/452`，但整个 `<path>` 或 `<line>` 引线元素缺失。

6. **[中等] MCP Protocol 无标注**  
   右侧两个绿色模块，Tool Interface 有标注（虽位置错），MCP 完全无标注，信息不对等。

7. **[轻微] 健康度环形图与 B- 语义不符**  
   `conic-gradient(#ffd93d 0deg 252deg` = 70% = C+ 范畴，但显示 B-。数据与图形割裂。

8. **[轻微] Spectrum Marker tooltip 未实现**  
   5个标记有 `title` 属性但无 JS 交互，用户看不到具体数值。

9. **[轻微] 响应式断点 900px 下 callout 全被隐藏**  
   对于复杂架构图，隐藏标注可能比显示更混乱。
