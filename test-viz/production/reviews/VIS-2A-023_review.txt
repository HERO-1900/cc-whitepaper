**缺陷:**

1. **语法错误 — 行70**
   ```css
   gap: 1px 2px;  /* ❌ CSS gap属性只接受单值或两个值，此处应删除 */
   ```
   `gap`是`row-gap`和`column-gap`的简写，此处`2px`被浏览器忽略为无效值。

2. **底部统计行单复数不一致 — 行318-321**
   ```html
   <div class="cell stat-row">
       <div class="coverage-num">1</div>
       <div class="coverage-label">Attack Covered</div>  <!-- 单数 -->
   </div>
   <div class="cell stat-row">
       <div class="coverage-num">3</div>
       <div class="coverage-label">Attacks Covered</div>  <!-- 复数 -->
   </div>
   ```
   L1行显示"1 Attack"但L2-L4显示"Attacks"，中文行始终用"Layers/Layer"，英文应统一。

3. **角落单元格视觉分隔不足 — 行86-92**
   表头角落格有渐变背景+斜角标题，但底部统计行角落区域是纯色空白，仅靠背景色区分，视觉边界模糊。

4. **hover缩放可能造成重叠 — 行155-158**
   ```css
   .cell.protected:hover {
       transform: scale(1.02);
       z-index: 10;
   ```
   多格同时hover时`scale`向各方向扩展，可能与相邻格重叠。`overflow: hidden`在`.matrix-container`或`.matrix`上缺失。

5. **空单元格斜线装饰层叠问题 — 行175-180**
   `.cell.empty::after`使用`transform: translate(-50%, -50%) rotate(-45deg)`，当背景通过gap透出边框色时斜线被边框切断，视觉不完整。

---

**修改建议:**

1. **行70** — 改为`gap: 1px;`

2. **行318-321** — 统一标签格式，建议所有统计格都用"Attacks Covered"（或全部用动态复数）

3. **底部统计行角落区域** — 添加与表头角落呼应的样式：
   ```css
   .cell.row-header[style*="border-top"] .attack-name-zh {
       color: var(--accent-primary);
   }
   ```
   或在HTML中给底部行角落格加上`style="background: linear-gradient(...)"`

4. **行154** `.matrix`添加`overflow: hidden;`或移除`scale`改用`box-shadow`强调

5. **空单元格斜线** — 考虑改用SVG或调整`z-index`确保斜线在边框之上，或移除斜线改为纯色背景强调"无覆盖"
