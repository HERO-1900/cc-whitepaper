旧客户端收到未知 surface 名称时，会丢弃未知项，降级为"少锁"而非"全坏"。非数组非布尔值会被捕获为 undefined，最终效果为"不锁"。

// z.preprocess 丢弃未知 surface，降级方向：少锁 const StrictPluginOnlyCustomizationSchema = z.preprocess((val) => { if (Array.isArray(val)) { return val.filter((s) => ['skills', 'agents', 'hooks', 'mcp'].includes(s)); } if (typeof val === 'boolean') return val; return undefined; }, z.union([z.boolean(), z.array(z.string())]).catch(undefined));
场景 A: 端到端管控
strict=true 叠加 strictKnownMarketplaces 时，管理员实现对企业环境的端到端控制。所有 4 个定制表面均被锁定，用户只能使用企业分发的内容。
strictPluginOnly=true marketplace gated
场景 B: 用户保留拒绝权
allowManagedMcpServersOnly=true 时，用户配置的 MCP 白名单被忽略，但用户仍然可以deny（拒绝）特定的 MCP 服务器。这是安全性与自主权的平衡设计。
managedMcpOnly=true deny retained
场景 C: 精准局部锁定
管理员可以只锁定特定表面。例如 ["skills","hooks"] 仅阻止 Skills 和 Hooks，Agents 与 MCP 仍可由用户自由定制。
skills locked hooks locked agents free mcp free
