优先级序列 Priority Sequence (1-9)

1. Policy (flag) — 命令行标志策略
2. Policy (managed) — 托管策略配置
3. Policy (enterprise) — 企业级配置
4. Policy (project) — 企业项目策略
5. Project Settings — 项目级设置
6. Local Project — 本地项目设置
7. User Settings — 用户全局设置
8. Default Settings — 系统默认值

覆盖规则 Override Rule

外层永远不可被内层覆盖。当配置冲突时，高优先级来源胜出。Policy 内部四个子层采用 first-source-wins 合并策略。
