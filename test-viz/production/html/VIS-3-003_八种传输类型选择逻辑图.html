MCP Transport Protocol Decision Map

八种传输协议选择逻辑 · Connection State Machine · Config Priority

决策树 / Decision Tree
传输对比 / Transport Comparison
stdio
Protocol: stdin / stdout
最适合: 本地子进程 MCP 服务器
stdio
  • 优点：简单、稳定、本地最常见
  • 缺点：仅限本地进程，无法远程
sse
Protocol: HTTP Server-Sent Events
最适合: 远程HTTP 遗留服务器
sse
  • 优点：兼容性好，易于穿透防火墙
  • 缺点：单向流，不支持现代流式能力
http
Protocol: Streamable HTTP
最适合: 现代远程 MCP (支持流式)
http
  • 优点：支持流式、无状态、可扩展
  • 缺点：需要服务器支持新版规范
ws
Protocol: WebSocket
最适合: 需要全双工实时通信
ws
  • 优点：全双工、低延迟、实时
  • 缺点：连接保持成本高，穿透复杂
sse-ide
Protocol: SSE + IDE Handler
最适合: VS Code / JetBrains 扩展
sse-ide
  • 优点：IDE集成优化，开箱即用
  • 缺点：绑定特定IDE环境
ws-ide
Protocol: WS + IDE Handler
最适合: IDE扩展 + WebSocket 场景
ws-ide
  • 优点：兼顾实时性与IDE特殊处理
  • 缺点：配置相对复杂
sdk
Protocol: SdkControlTransport
最适合: 同进程 SDK 直接控制
sdk
  • 优点：零网络开销，直接调用
  • 缺点：仅适用于SDK内部嵌入
claudeai-proxy
Protocol: ClaudeAuthProvider
最适合: claude.ai 代理路由
claudeai-proxy
  • 优点：云端代理，URL重写安全
  • 缺点：依赖 claude.ai 平台
