词法分析与解析

Lexical Analysis & Parsing
bashParser.ts
纯 TypeScript 实现的词法器与解析器。将输入字符串转换为 Token 流，再构建为 AST 抽象语法树。
◆ 处理流程
  • 输入字符串 → Token 流
  • Token 流 → AST 节点
  • 支持 12 种 AST 节点类型
◆ 安全机制
  • 解析失败触发 FAIL-CLOSED
  • 标记为 too-complex 状态
  • 防止解析器被绕过
Fail-Closed Guard

AST 安全分析

AST Security Analysis
bashAnalyze.ts
遍历 AST 提取命令列表，递归处理复合结构。白名单机制：只放行的认识的节点类型。
◆ 递归处理结构
  • Pipeline 管道结构
  • List 命令列表
  • Compound 复合命令
◆ 白名单机制
  • 未知节点类型 → too-complex
  • 无法解析结构 → FAIL-CLOSED
  • 显式放行已知类型
Fail-Closed Guard
3

24项安全检查

Security Checks
checkBashCommand
细粒度命令安全检查。每项检查返回 allow / ask / deny 三种决策之一。
◆ 典型检查项
  • rm -rf / 危险删除检查
  • git push --force 检查
  • chmod 777 权限提升检查
◆ 决策逻辑
  • 任一项返回 deny → 拒绝执行
  • 多项 ask → 汇总确认
  • 全部 allow → 放行
Deny = Immediate Block
4

权限系统

Permission System
hasPermissionsToUseTool
最终权限判定层。整合全局规则、工具级权限与绕过免疫区域。
◆ 检查层级
  • 全局 allow/deny 规则
  • 工具级权限检查
  • bypass-immune 保护区域
◆ 最终判定
  • 自动执行 (安全命令)
  • 用户确认 (可疑命令)
  • 拒绝执行 (明确危险)

?

Ask / User Confirm
可疑命令或 too-complex 标记，需用户显式确认
✕

Deny / Block
明确危险命令触发安全检查拒绝，执行被阻止
