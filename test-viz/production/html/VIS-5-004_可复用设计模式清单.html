可复用设计模式清单

Reusable Design Patterns from Claude Code

①
在等待时间里藏工作 Speculative Execution
找到"必须等待X"的地方 → 等待期间做不依赖X的工作
src/main.tsx 启动序列 (API等待期间加载模块)
Promise.all 并行化 可回滚
②
Token是一等公民 Tokens as First-Class Citizens
识别消耗 → 测量 → 优化 → 监控
omitClaudeMd / CacheSafeParams / 6层压缩
Prompt精简 模型分级 Cache对齐
⚠️ 多AI实例共享cache时参数必须完全相同，否则成本暴涨45倍
③
AI设计成可组合模块 Composable AI Modules
统一执行引擎 + 不同参数配置
query() @ src/services/claude.ts (7种querySource)
独立AbortController 限制工具集
④
多层防线 Defense in Depth
硬编码禁止 → 配置拒绝 → 操作自检 → 配置允许 → 默认保守
9步权限状态机 (permissions.ts) / bypass-immune / iron gate
fail-closed bypass-immune 退出码语义
⑤
可观测性是产品的一部分 Observability by Design
设计功能时同时设计其可观测性
27个事件生命周期 / Span层级 / Perfetto trace
日志即代码 同等优先级
⑥
Hooks扩展点系统 Hooks Extension Points
事件节点 + 退出码语义 + 安全默认 + 来源标记 + 异步支持
PreToolUse / PostToolUse / UserPromptSubmit 等8个事件
退出码: 0=允许 2=给AI看
⑦
stat-before-content优化 Metadata-First Optimization
先用廉价操作判断是否需要昂贵操作
fileHistory的stat()先查再读 / mtime去重
O(1)元数据检查 避免重复IO
⑧
三级降级策略 Three-Tier Degradation
完整方案 → 精简方案 → 最小可用方案
Git bundle --all → HEAD → squashed-root
Graceful Degradation 可用性保障
⑨
路径哈希命名 Path-Based Hash Naming
用路径哈希(非内容哈希)作为缓存/备份键
SHA256(filePath).slice(0,16)@vN
O(1)查找 空间换时间
⑩
拒绝而非截断 Reject, Don't Truncate
超限时抛错而不是静默截断
parseDeepLink的q参数超5000字符直接拒绝
语义完整性 安全优先
