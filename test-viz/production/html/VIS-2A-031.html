流式渲染热路径

Streaming Hot Path: API 响应以 chunk 形式流入 AssistantMessage，导致从 ScrollBox 到 MessageList 的整条路径频繁重渲染。

焦点管理

FocusProvider 维护焦点栈。当 PermissionDialog 弹出时，它会自动夺取焦点，用户确认后焦点归还至 InputBox。

错误边界

ErrorBoundary 独立于主树右侧，捕获任何子树的渲染错误，防止整个终端 React 应用崩溃并退出备用屏幕。

数据流方向

InputBox → 用户消息 → API 调用 → stream chunks → state update → MessageList → AssistantMessage (re-render per chunk)。
